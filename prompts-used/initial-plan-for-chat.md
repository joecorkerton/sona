Looking through the README, task-info and market-research, help me plan out the first set of functionality in the app - building a basic chat interface for both group and 1-1 chats. Let's start with a plan of the high level functionality and approach, and then go into splitting the work up into smaller independent chunks. The chat is intentionally left simple to start with, but with an architecture that lets it easily expand down the line. Initially lets cover the use case of simple 1-1 and group chats without any replies or thread capabilities, but with the possibility of being able to add those easily in the future. Chat data should be stored in postgres. To get users to signup, have a functionality of a share link, which they can open and choose a username and start chatting. For this POC, we won't cover authentication and passwords as it's not important for now. Users will be identified by their username, so if they re-access with the same username they load in the same chat history. We should use liveview for all of the interfaces, and prioritise mobile UIs while still functioning on desktop
